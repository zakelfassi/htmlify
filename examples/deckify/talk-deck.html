3.0The problem

Terminal answers fail twice

Ephemeral

  • Scrollback is a buffer, not a record
  • Close the tab, lose the analysis
  • Context compaction eats the middle of long sessions
  • Nothing to send to a teammate or a meeting

Unreadable at length

  • Comparisons become pipe-table ASCII art
  • Architecture has no spatial layout in a stream
  • No side-by-side, no jump links, no collapse
  • 2,000-word markdown is a wall, not a document

Short answers are fine in the terminal. The failure starts exactly when the answer becomes worth keeping.

Plate 03 / 14Talk-Deckstdout, made permanent
Two distinct failures, and people usually only name the first. Ephemerality is obvious — scrollback dies. The subtler one is that markdown stops working as a reading surface at length: a six-option comparison or an incident timeline has a SHAPE, and a token stream flattens it. Punchline: the better the answer, the worse the medium fits.

4.0The thesis

Answers with shape deserve a document.

If the reader needs to compare, navigate, review, archive, or hand the answer to someone else — it should be one self-contained HTML file, not a scroll of markdown. The browser is the most widely deployed document reader on earth, and it is already on your machine.

Plate 04 / 14Talk-Deckstdout, made permanent
This is the one-sentence takeaway; say it twice. "Shape" is the operative word — comparative, spatial, navigable, reviewable. Pre-empt the objection: this is not "make everything a webapp." No server, no build step, no framework. One file, file:// protocol, double-click. The terminal keeps everything that is short and linear; documents get the rest.

5.0Mental model

Evidence → author → validate. One file out.

Plate 05 / 14Talk-Deckstdout, made permanent
Three verbs, in order, no skipping. Evidence first: the skill reads the repo, diffs, logs, and docs before designing anything — artifacts assert things, so claims need receipts. Author: exactly one file, all CSS and JS inline. Validate: a real validator with profiles, and zero errors is the bar for shipping — same posture as tests before merge. The orange arrow is the only product: a file.

7.0The skill family

Two skills, one contract

htmlify — operator artifacts

  • operator-brief · status-report — what happened, what is next
  • build-plan · implementation-map — phases, files, data flow
  • pr-review-packet · release-brief — diff tour, deploy proof
  • incident-report · decision-brief — timeline, options, tradeoffs
  • explainer · prototype / editor — concepts, clickable flows

deckify — presentation artifacts

  • talk-deck — speaker notes + run-of-show (this file)
  • workshop-deck — exercises, labs, checkpoints
  • essay-deck — deck plus long-form guide
  • demo-deck — live demos with fallbacks
  • launch-deck · teaching-guide — narrative, PDF-first

Shared rules: smallest mode that fits, evidence first, self-contained file, validate before shipping.

Plate 07 / 14Talk-Deckstdout, made permanent
Mode selection is the actual skill here: the instruction is "pick the smallest artifact mode that fits," which prevents both over-production (a deck for a yes/no question) and under-production (a markdown wall for an incident review). htmlify covers operator work; deckify extends it with notes, timing, and guide output for anything you present. Mention the meta-proof: this deck is deckify's own talk-deck mode, validated by the repo's validator.

8.0Self-containment is a security posture

Generated HTML is untrusted until validated

Banned in every profileWhy
<script src> · CDNsno remote code, ever
inline on* handler attributesscript goes in one auditable block
javascript: URLs · meta refreshno sneaky execution or redirects
iframe · object · embed · baseno foreign documents inside yours
external fonts, images, css url()no phone-home, works offline forever
Plate 08 / 14Talk-Deckstdout, made permanent
Key reframe: self-containment is not an aesthetic preference, it is the threat model. A model-generated file that can load remote code is an exfiltration vector; a file that cannot reach the network is just a document. The validator enforces this mechanically — banned constructs are errors, and an artifact with errors does not ship; the system falls back to plain local HTML instead. Profiles ratchet: rich has zero script, app allows one inline block, deck adds the structural contract this very file passed.

9.0The annotation loop

The document talks back

Plate 09 / 14Talk-Deckstdout, made permanent
This is the feature that makes artifacts a loop instead of a dead end. You review the document the way you review a PR: select a passage, leave a note anchored to it. Export produces one markdown bundle — quote, comment, anchor — which is exactly the format an agent ingests well. Paste it back, the agent revises, the orange edge closes the loop. The artifact becomes the shared workspace between you and the model, replacing "scroll up and describe what you meant."

11.0Failure modes

When to stay in the terminal

Answer shapeRight channelWhy
Short, linear, command-likestdoutyou will act, not re-read
Yes/no plus one reasonstdouta file is ceremony
Mid-loop iteration, half-formedstdoutartifacts assert; drafts should not
Comparison, plan, timeline, reviewartifactshape needs layout
Anything headed to a meeting or archiveartifactpermanence is the point

Rule of thumb: if you would never scroll back to it, do not make a file out of it.

Plate 11 / 14Talk-Deckstdout, made permanent
Credibility slide — argue against your own tool. Most agent output should STAY in the terminal: command sequences, quick lookups, mid-loop iteration. An artifact for a yes/no answer is ceremony, and worse, artifacts carry authority — a half-formed draft typeset like an engineering plate reads as more settled than it is. The skill encodes this as "smallest mode that fits," where the smallest mode is often markdown. Land the rule of thumb verbatim.
